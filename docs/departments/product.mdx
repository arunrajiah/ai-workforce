# Ship a user-research + PRD agent
> Replaces: research contractors, Typeform, Amplitude, PRD busywork. Ship in: an afternoon.

This guide assembles an open-source product stack: a deep-research agent, a survey/feedback tool, and a product-analytics platform. The LLM runs multi-source research and drafts PRDs from the signals you collect.

## What you'll build
- [GPT Researcher](https://github.com/assafelovic/gpt-researcher) — autonomous deep research.
- [Formbricks](https://github.com/formbricks/formbricks) — in-app surveys and feedback.
- [PostHog](https://github.com/PostHog/posthog) — product analytics and session insight.

## Prerequisites
An LLM key and Docker. See [`/quickstart`](/quickstart) for the shared foundation.

## Step 1 — Stand up deep research (GPT Researcher)
```bash
git clone https://github.com/assafelovic/gpt-researcher.git
cd gpt-researcher
cp .env.example .env   # set your LLM endpoint (OpenAI-compatible) + TAVILY_API_KEY
docker compose up -d
```
GPT Researcher serves on port `8000`. Ask it to research a market/competitor and it returns a cited report.

## Step 2 — Add user feedback (Formbricks)
```bash
git clone https://github.com/formbricks/formbricks.git
cd formbricks
cp .env.example .env   # set NEXTAUTH_SECRET, ENCRYPTION_KEY, DATABASE_URL
docker compose up -d
```
Formbricks serves on port `3000`. Embed micro-surveys to capture in-context user signals.

## Step 3 — Add product analytics (PostHog)
```bash
git clone https://github.com/PostHog/posthog.git
cd posthog
docker compose -f docker-compose.hobby.yml up -d
```
PostHog serves on port `8000` — remap if it collides with GPT Researcher. Add the snippet to your app.

## Step 4 — Wire in the AI
GPT Researcher reads its LLM config from `.env` — set any OpenAI-compatible endpoint (`LLM_BASE_URL` / `LLM_MODEL`) to power the research runs. Chain it with an n8n/Activepieces flow: pull Formbricks responses + PostHog funnels → LLM call → draft a PRD. For local/offline research, point GPT Researcher at [Ollama](https://github.com/ollama/ollama).

## Ship it
Put [Caddy](https://caddyserver.com) in front of each service for automatic HTTPS. See [`/quickstart`](/quickstart).
