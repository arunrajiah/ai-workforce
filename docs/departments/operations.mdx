# Ship a workflow-automation backbone
> Replaces: Zapier, Retool workflows, Asana, Notion. Ship in: a day.

This guide stands up an open-source ops backbone: a no-code automation engine, a scripts-as-workflows platform, a project tracker, and a docs/wiki tool. The LLM sits inside your workflows to summarize, classify, and route.

## What you'll build
- [n8n](https://github.com/n8n-io/n8n) — no-code automation.
- [Windmill](https://github.com/windmill-labs/windmill) — scripts-as-workflows.
- [Plane](https://github.com/makeplane/plane) — project and issue tracking.
- [AppFlowy](https://github.com/AppFlowy-IO/AppFlowy) — docs and wiki.

## Prerequisites
An LLM key and Docker. See [`/quickstart`](/quickstart) for the shared foundation.

## Step 1 — Stand up automation (n8n)
```bash
docker run -d --name n8n -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n
```
n8n serves on port `5678`. This is the hub the rest of the stack hangs off.

## Step 2 — Add scripts-as-workflows (Windmill)
```bash
mkdir windmill && cd windmill
curl -LO https://raw.githubusercontent.com/windmill-labs/windmill/main/docker-compose.yml
docker compose up -d
```
Windmill serves on port `8000`. Use it for typed Python/TS jobs n8n triggers.

## Step 3 — Add project tracking (Plane)
```bash
mkdir plane && cd plane
curl -fsSL https://raw.githubusercontent.com/makeplane/plane/master/setup.sh | sh
```
Plane serves on port `80`. Set secrets in `plane.env` generated by the installer.

## Step 4 — Add docs/wiki (AppFlowy)
```bash
git clone https://github.com/AppFlowy-IO/AppFlowy-Cloud.git
cd AppFlowy-Cloud
cp deploy.env .env   # set GOTRUE, Postgres, and app secrets
docker compose up -d
```
AppFlowy-Cloud serves on port `80`. Point the desktop app at your server URL.

## Step 5 — Wire in the AI
In n8n, add an AI node pointed at any OpenAI-compatible LLM endpoint (`LLM_BASE_URL` / `LLM_MODEL`) to triage inbound requests, summarize threads, and create Plane issues automatically. Windmill scripts can call the same endpoint for heavier processing. For local/offline, point the nodes at [Ollama](https://github.com/ollama/ollama).

## Ship it
Put [Caddy](https://caddyserver.com) in front of each service for automatic HTTPS. See [`/quickstart`](/quickstart).
