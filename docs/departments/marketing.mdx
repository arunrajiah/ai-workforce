# Ship a content + SEO engine
> Replaces: Jasper, Mailchimp, Buffer, Ahrefs, a chunk of your content agency. Ship in: a day.

This guide assembles an open-source marketing stack: an agent builder for content and SEO workflows, an email sender for newsletters, a social scheduler, product analytics, and a scraper for SEO research. The LLM lives inside Dify and drives the content pipeline.

## What you'll build
- [Dify](https://github.com/langgenius/dify) — build content and SEO agents/workflows.
- [listmonk](https://github.com/knadh/listmonk) — self-hosted newsletter/email.
- [Postiz](https://github.com/gitroomhq/postiz-app) — schedule and publish social posts.
- [PostHog](https://github.com/PostHog/posthog) — product and marketing analytics.
- [Firecrawl](https://github.com/mendableai/firecrawl) — SERP and competitor research.

## Prerequisites
An LLM key and Docker. See [`/quickstart`](/quickstart) for the shared foundation.

## Step 1 — Stand up the agent builder (Dify)
```bash
git clone https://github.com/langgenius/dify.git
cd dify/docker
cp .env.example .env   # set SECRET_KEY, DB creds
docker compose up -d
```
Dify serves on port `80`/`3000` via nginx. Create your first content agent in the console.

## Step 2 — Add email (listmonk)
```bash
mkdir listmonk && cd listmonk
curl -LO https://raw.githubusercontent.com/knadh/listmonk/master/docker-compose.yml
docker compose up -d
```
listmonk serves on port `9000`. Set your SMTP creds in the admin UI or `config.toml`.

## Step 3 — Add social scheduling (Postiz)
```bash
docker run -d --name postiz -p 5000:5000 \
  -e MAIN_URL="https://post.example.com" \
  -v postiz_config:/config \
  ghcr.io/gitroomhq/postiz-app:latest
```
Postiz serves on port `5000`. Connect your social accounts via `.env`/OAuth in settings.

## Step 4 — Add analytics (PostHog)
```bash
git clone https://github.com/PostHog/posthog.git
cd posthog
docker compose -f docker-compose.hobby.yml up -d
```
PostHog serves on port `8000`. Drop the snippet on your site to track content performance.

## Step 5 — Add SEO research (Firecrawl)
```bash
git clone https://github.com/mendableai/firecrawl.git
cd firecrawl && cp apps/api/.env.example .env
docker compose up -d
```
Firecrawl API on port `3002` — feed competitor pages/SERPs into your Dify agents.

## Step 6 — Wire in the AI
In Dify, open **Settings → Model Provider** and add any OpenAI-compatible provider (set the base URL and model). Build a workflow: Firecrawl research → outline → draft → push to listmonk/Postiz. For fully local generation, add [Ollama](https://github.com/ollama/ollama) as the provider (`http://localhost:11434/v1`).

## Ship it
Put [Caddy](https://caddyserver.com) in front of each service for automatic HTTPS. See [`/quickstart`](/quickstart).
